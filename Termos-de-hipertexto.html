Termos de hipertexto

Este é um glossário de termos usados ​​no WWW projeto. Na maioria dos casos, seu uso corresponde ao uso de conversão em círculos de hipertexto.

Âncora

Uma área dentro de um conteúdo de um nó que é a origem ou destino de um link. A âncora pode ser todo o conteúdo do nó. Normalmente, clicar com o mouse em uma área de âncora faz com que o link seja seguido, deixando a âncora na extremidade oposta do link exibido. As âncoras tendem a ser destacadas de maneira especial (sempre, ou quando o mouse está sobre elas) ou representadas por um símbolo especial. Uma âncora pode, e muitas vezes corresponde, a todo o nó. (também conhecido como "span", "região", "botão" ou "extensão").
Anotação
A vinculação de um novo comentário nó para um nó existente. Se os leitores puderem anotar nós, eles poderão fornecer feedback imediatamente se as informações forem enganosas, desatualizadas ou simplesmente erradas. Assim, a qualidade da informação na web pode ser melhorada. (Mais...)
Autoria
A term for the process of writing a document. "Authoring" seems to have come into use in order to emphasise that document production involved more than just writing.
Voltar link
Um link em uma direção implicava a existência de um link explícito na outra direção. Ver: Construindo backlinks
Navegador
Um programa que permite pessoa ler hipertexto. O navegador oferece alguns meios de visualizar o conteúdo de nós, e de navegação de um nó para outro.
Botão
Um âncora que é a fonte de um link. Muitas vezes, mas nem sempre, representado na tela para parecer um botão de pressão.
Cartão
Um termo alternativo para nó em um sistema (por exemplo, HyperCard, Notecards) no qual o tamanho do nó é limitado a uma única página de tamanho limitado.
Cliente
Um programa que solicita serviços de outro programa. Normalmente, o navegador é um cliente de um servidor de dados.
Base de dados
Usamos isso vagamente como um termo para uma coleção de nós Imaginamos que as informações de gerenciamento de um deles sejam mantidas em um só lugar e todas acessíveis pelo mesmo servidor. Links fora isso são "externos", e aqueles dentro são "internos". Não insinuamos nada sobre como as informações escoradas serão armazenadas.
Daemon
A program which runs independently of, for example the navegador. Daemons podem executar várias tarefas de gerenciamento, como building indexes, visões gerais, e back-links. No unix, "daemon" é usado para " servidor", porque os servidores normalmente são executados de forma independente.
Documento
Um termo para nó em alguns sistemas (por exemplo, Intermedia). Às vezes usado por outros como um termo para uma coleção de nós em tópicos relacionados, possivelmente armazenados ou distribuídos como um. O termo preferido na documentação do W3.
Domínio
Usamos isso especificamente para uma unidade de proteção. Poderia corresponder a um database, e nesse caso seria um termo melhor (menos vago) para isso.
Externo
UMA link para um nó em um banco de dados diferente. Ver Base de dados
Hospedeiro
A computer on a network. We use this term rather than the term " nó " que é frequentemente usado para um documento em um hipertexto rede.
Hipermídia
MultiMedia Hipertexto. HyperMedia e HyperText tendem a ser usados ​​livremente no lugar um do outro. Mídia que não seja texto normalmente inclui gráficos, som e vídeo ( Mais)
Interno
Uma link para um nó in the same database. Ver base de dados.
Link
Uma relação entre dois âncoras, armazenados no mesmo ou em diferentes base de dados. Ver "Interno" e "External".
Navegação
O processo de passar de um nó para outro através do hipertexto rede. Isso normalmente é feito seguindo links. Várias características de um determinado navegador pode tornar isso mais fácil. Isso inclui manter um histórico de onde o usuário esteve e desenhar diagramas de links entre nós próximos (Mais...)
Nó
Uma unidade de informação. Também conhecido como quadro (KMS), cartão (Hypercard, Notecards). Usado com este significado especial em círculos de hipertexto: não confunda com "nó" que significa "host de rede". Para benefício do usuário, usamos o termo " documento" pois este é o termo mais próximo fora do mundo do hipertexto.
Proteção
A prevenção de usuários não autorizados de ler ou gravar um determinado dado. Também conhecido como "autenticação", "controle de acesso", etc. (More...)
Caminho
Um conjunto ordenado de nós ou âncoras que representam uma sequência na qual um rede pode ser lido. Um caminho pode representar a sequência que um leitor realmente usou ou pode ser uma sequência recomendada ao leitor pelo autor.
Leitor
Usamos este termo para a pessoa que navega, para distingui-la do programa( navegador) que ele(a) usa.
Servidor
Um programa que presta um serviço a outro, conhecido como cliente. Em um hipertexto sistema, um servidor fornecerá informações de hipertexto a um navegador. Veja também: demônio.
Rastreamento
A descoberta automática de nós por navegação. Exemplos podem ser encontrar todos os nós dependentes de outro nó, todas as pessoas interessadas em um determinado nó, todos os módulos que usam um determinado módulo. Outro exemplo é um rastreamento que começa com mais de um nó, como para encontrar um nó em comum entre dois grupos ou um caminho ligando dois nós.
Topologia
A conectividade permitida entre nós, âncoras e links: por exemplo, mapeamentos 1-1 ou muitos-1. (Mais...)
Controle de versão
O armazenamento e gerenciamento de versões anteriores de uma informação, para segurança, diagnóstico e interesse. Isso é importante quando muitos usuários têm permissão para editar o mesmo material (Mais...)
Rede
Um conjunto de nós interligados por links. Often, the set of all the nodes which are interconnected. See also Topologia.
